## 프로젝트 개요

본 프로젝트는 건국대학교 학생들의 졸업요건을 학생 개인의 이수 현황에 맞게 시각화하기 위한 
맞춤형 졸업요건 분석 서비스 Zol.ver의 데이터 확보를 목표로 합니다.

이를 위해 PDF 형태로 제공되는 비정형 학사 데이터를 신뢰 가능한 데이터 자산으로 변환하는
데이터 파이프라인을 설계하고 있습니다.
파이프라인은 PDF → Text → 구조화된 JSON으로 이어지며, 각 처리 단계에서 발생하는 input/output 관계와
데이터 품질 이슈를 로그 관점에서 추적하고 Data Lineage로 확장 가능한 구조를 갖습니다.

현재는 초기 설계 및 실험 단계로, LLM 기반 구조화 과정에서 발생하는 비결정적 출력 문제를 중심으로
파이프라인의 안정성, 재현성, 품질 관리 구조를 설계하고 있습니다.

## Directory Design Rationale
### data/

원천 데이터 보관 영역입니다.
파이프라인 입력 데이터와 코드 영역을 명확히 분리하여 재현성과 추적성을 확보합니다.

### docs/data_pipeline/

데이터 구조화 및 처리 과정에서 발생한 이슈를 문서로 정의합니다.

### notebooks/

실험 및 PoC 단계의 결과를 보존합니다.
운영 코드(src)와 분리하여 실험 코드가 서비스 코드에 영향을 주지 않도록 구성합니다.

### src/

서비스 배포를 고려한 데이터 파이프라인 코드 영역입니다.
각 처리 단계를 역할 단위로 분리하여 유지보수성과 확장성을 확보합니다.

### sources/
데이터 입력 계층 (PDF ingestion)

### parsing/
포맷 해석 계층 (PDF → Text)

### preprocessing/
텍스트 정제 및 전처리

### quality/
임베딩 기반 의미 검증, 규칙 기반 검증, 안정성 및 재현성 평가를 담당

### lineage/
데이터 변환 과정에서 발생하는 input/output 관계를 기록하고 추적

### database/
DB schema 설계 및 파이프라인 결과 저장을 위한 DB 연결 및 세션 관리

### utils/
전역적으로 재사용 가능한 공통 유틸리티 관리
로깅, 설정, 해시, ID 생성 등 전역적으로 재사용 가능한 공통 유틸리티
