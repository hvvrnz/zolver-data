## 프로젝트 개요

본 프로젝트는 건국대학교 학생들의 졸업요건을 학생 개인의 이수 현황에 맞게 시각화하기 위한  
맞춤형 졸업요건 분석 서비스 **Zol.ver**의 데이터 확보를 목표로 합니다.

이를 위해 PDF 형태로 제공되는 비정형 학사 데이터를  
**신뢰 가능한 데이터 자산**으로 변환하는 데이터 파이프라인을 설계하고 있습니다.

### Data Pipeline Overview

파이프라인은 다음과 같은 흐름을 따릅니다.

- **PDF → Text → 구조화된 JSON**
- 각 처리 단계의 input/output 관계 및 데이터 품질 이슈를 로그 관점에서 추적
- 전체 흐름을 **Data Lineage**로 확장 가능한 구조로 설계

현재는 초기 설계 및 실험 단계로,  
LLM 기반 구조화 과정에서 발생하는 **비결정적 출력 문제**를 중심으로  
파이프라인의 안정성, 재현성, 품질 관리 구조를 검증하고 있습니다.

---

## Directory Design Rationale

각 디렉터리는 데이터 파이프라인 내에서 명확한 역할을 가지며,  
재현성·추적성·확장성을 고려해 분리 설계되었습니다.

### data/
**원천 데이터 보관 영역**

파이프라인 입력 데이터와 코드 영역을 명확히 분리하여  
재현성과 추적성을 확보합니다.

---

### docs/data_pipeline/
**문제 정의 및 설계 문서**

데이터 구조화 및 처리 과정에서 발생한 이슈를  
코드 이전 단계에서 문서로 정의합니다.

---

### notebooks/
**실험 및 PoC 기록**

LLM 실험 및 구조화 검증 결과를 보존하며,  
운영 코드(`src`)에 영향을 주지 않도록 분리합니다.

---

### src/
**데이터 파이프라인 핵심 코드**

서비스 배포를 고려한 코드 영역으로,  
각 처리 단계를 역할 단위로 분리하여 유지보수성과 확장성을 확보합니다.

- #### sources/
  데이터 입력 계층 (PDF ingestion)

- #### parsing/
  포맷 해석 계층 (PDF → Text)

- #### preprocessing/
  텍스트 정제 및 전처리

- #### quality/
  임베딩 기반 의미 검증, 규칙 기반 검증,  
  안정성 및 재현성 평가를 담당

- #### lineage/
  데이터 변환 과정에서 발생하는  
  input/output 관계를 기록하고 추적

- #### database/
  DB schema 설계 및 파이프라인 결과 저장을 위한  
  DB 연결 및 세션 관리

- #### utils/
  전역적으로 재사용 가능한 공통 유틸리티 관리
