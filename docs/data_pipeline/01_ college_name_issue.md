### Issue: 학과 구조화 시 단과대학명 분절 인식 오류 발생

PDF -> 텍스트 변환 과정에서 단과대학 이름이 줄바꿈 단위로 분리되어 
LLM이 "인문사회융합대학"을 "인문", "사회", "융합", "대학"처럼 독립된 섹션으로 오판하는 문제가 발생

#### 예시 입력:
{raw_text} : 졸업에 필요한 최저 이수 학점 수\n대학\n학과\n교양\n전공\n다전공\n학생\n졸업이수\n학점\n전필\n전선\n계\n디자인\n대학\n산업디자인학과\n35\n14\n58\n72\n40\n132\n실내디자인학과\n35\n13\n56\n69\n40\n132\n패션디자인학과\n35\n2\n70\n72\n40\n132\n시각영상디자인학과\ ...


#### 예시 출력 문제:
 {
      "college_name": "산업디자인학과",
      "departments": [
        {
          "department_name": "",
          "credits": {
            "general_education": 58,
            "major_required": 14,
            "major_elective": 0,
            "major_total": 72,
            "double_major": 40,
            "graduation_total": 132
          }
        },
        {
          "department_name": "실내디자인학과",
          "credits": {
            "general_education": 56,

        또는

        
      "college_name": "인문",
      "departments": [
        {
          "department_name": "경영학과",
          "credits": {

#### 원인:
1. PDF 텍스트 추출 시 개행 단위 분리
2. LLM이 복합명사를 줄 단위로 오해하는 패턴 오류

#### 해결 방향:
1. parsing (완료)
단과대학명 후보 영역의 개행 문자 제거 

2. preprocessing
개행 제거 후에도 애매한 경우
테이블 형식으로 구조화 시도 및 
embedding 기반 유사도 분석으로 복합 명사 복원
(예: 디자인 + 대학 → 디자인대학)

3. LLM 프롬프트 (임시) (현재는 제거 함.)
단과대학명은 공백 없는 하나의 문자열이며
‘대학’으로 끝나는 복합 명사라는 규칙 적용
전처리 및 검증 로직 안정화 후 제거 예정

4. JSON 후처리 (예정)
단과대학명 검증 및 보정

#### 설계 원칙 준수
- 도메인 규칙은 코드에서 처리
- LLM은 구조 변환 역할만 담당
- 프롬프트 제약은 입력 품질 보완을 위한 임시 조치

#### Note
본 이슈는 초기 설계 단계의 안정화를 위한 기록이며,
후속 설계에서는 전처리/검증 중심으로 더 단순화될 예정.
