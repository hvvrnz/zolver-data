### JSON 출력 결과 앞 뒤 부연 설명이 붙어 출력 데이터 품질 저하 발생

LLM이 "부연 설명은 하지 말고 반드시 유효한 JSON만 출력하세요"라는 명령을 무시하고, 설명을 붙여 출력하는 문제가 발생한다. 어떨 땐 안 나오고 어떨 땐 나옴.

#### 예시 입력:
당신은 데이터 구조화 전문가입니다.
아래 텍스트를 JSON으로 변환하세요.

단과대학 이름은 공백 없이 합쳐진 형태여야 합니다.
예: "인문사회융합대학", "과학기술대학", "디자인대학"
"인문", "사회", "융합", "대학"처럼 개별 단어로는 절대로 분리하지 마십시오.
"응용 화학 과", "스포츠 건강학 과"처럼 공백을 포함하지 마십시오.


부연 설명은 하지 말고 반드시 유효한 JSON만 출력하세요.
형식 예시는 다음과 같습니다:

#### 예시 출력 문제:
아래는 입력된 텍스트를 위 JSON 구조에 맞게 구조화한 결과입니다.

```json
{
  "year": "2021",
  "universities": [
    {
      "college": "인문사회융합대학",
      "department
      ...
    }
  ]
}
```
위의 JSON 형태는 2021년도 건국대학교 요람에서 추출된 데이터를 구조화한 결과입니다. 각 대학과 학과에 대한 정보가 포함되어 있습니다.

#### 원인:
1. llm은 parser 및 사람처럼 “JSON 규칙을 정확히 지키려고” 동작하는 게 아니라 이럴 때 사람들이 보통 어떤 글을 쓰지?'를 확률적(확률 기반)으로 예측함.

어떤 상황에서는 JSON 끝에 "원인은..." 같은 말이 자연스러워 보여서 붙임
어떤 상황에서는 JSON만 쓰는 게 자연스러워 보여서 깔끔하게 끝남
즉, 일관된 포맷을 강제할 수 없음

#### 해결 방향:
1. text_to_json_with_llm 출력 결과의 텍스트를 정규표현식으로 정제 (가장 바깥쪽 { } , JSON의 출력결과만 나오도록)
