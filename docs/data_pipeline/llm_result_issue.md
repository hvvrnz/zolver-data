### Issue: JSON 출력 결과 앞 뒤 부연 설명이 붙어 출력 데이터 품질 저하 발생

LLM이 "부연 설명은 하지 말고 반드시 유효한 JSON만 출력하세요"라는 명령을 무시하고, 설명을 붙여 출력하는 문제가 발생한다. 
(어떨 땐 잘 안 나오고 어떨 땐 나오기도 한다.)

#### 예시 입력:
당신은 데이터 구조화 전문가입니다.
아래 텍스트를 JSON으로 변환하세요.

형식 예시는 다음과 같습니다:
{JSON_SCHEMA_EXAMPLE}

텍스트:
{raw_text}


#### 예시 출력 문제:
하단 부분 출력 예시

주의사항: 
이 텍스트는 모든 학생의 학점 이수를 표시하고 있지 않으며, 각 과에 따라 다르게 나타낼 수 있습니다.
데이터 구조화에는 다양한 방법이 있으며, 본 예시는 단순히 데이터를 JSON으로轉換한 것일 뿐이므로, 필요에 따라 데이터를 추가 편집하실 수 있습니다.

또는

위 예시는 모든 데이터를 정확하게 변환한 것처럼 보이지만, 모든 데이터가 정확한지는 확인하지 못ました. 만약에 정확한 변환을 하기 위해서는 더 많은 작업이 필요할 것 같습니다.



#### 원인:
1. llm은 parser 및 사람처럼 “JSON 규칙을 정확히 지키려고” 동작하는 게 아니라 이럴 때 사람들이 보통 어떤 글을 쓰지?'를 확률적(확률 기반)으로 예측
  1-1. 어떤 상황에서는 JSON 끝에 "원인은...", "주의사항은..." 같은 말이 자연스러워 보여서 붙임
  1-2. 어떤 상황에서는 JSON만 쓰는 게 자연스러워 보여서 깔끔하게 끝남
  -> 즉, 일관된 포맷을 강제할 수 없음

#### 해결 방향:
1. Rule-Based 매핑(진행중)
  1-1. text_to_json_with_llm()의 출력 결과의 텍스트를 정규표현식으로 정제한다. (가장 바깥쪽 { } , JSON의 출력결과만 나오도록) 
2. Prompt에 규칙 추가 (완료) -> 미해결
